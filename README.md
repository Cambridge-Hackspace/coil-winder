# coil-winder

Welcome to the [Cambridge Hackspace](https://www.cambridgehackspace.com)
Coil Winder Project! This repository is essentially where I'll be tracking and
documenting progress on an open-source axial flux coil winder project.

The entirety of this project is released under the [MIT License](https://mit-license.org/).
We humbly request, but do not require, that you refrain from using this
project for Evil.

## Background

I want to build an open source flip-dot display. But in order to do that,
I'm going to need two electromagnets for each flip dot. If I end up even making
a small 50x50 display, I suddenly need 5,000 electromagnets and I'm much
too lazy to do that all by hand. So I'm procrastinating on that project by
building this beforehand.

I'm also doing this as an excuse to work a little bit more with microcontrollers,
motors, and small electronic components. Quite frankly, my soldering skills could
use a little work. In addition, I have a few goals I'm trying to stick to
throughout the project--but I'm reserving the right to make exceptions at any
time.

1. I'd like this project to remain as DIY as possible, to the extent possible
   through tools commonly found in a makerspace. So I'll want to make use of
   3D printers, CNC machines, etc. where I can, but I'm certainly not going to
   go so far as to fabricate my own resistors or whatever.
2. I'm hoping to use parts I find around the Hackspace. We've got a lot of old
   microcontrollers tucked away in boxes, rarely ever seeing the light of day.
   I want to try to use the bottom of the barrel so that we're contributing
   less to e-waste.
3. I want to stick with a code-first approach. So this mostly pertains to the
   CAD stuff I think, but basically I want to make sure that I'm sticking to
   FLOSS where I can, and I want to make sure I can maintain good git diffs.

So this might mean that some of the design decisions or components I use are
older, or not as optimal--and in the vein of the hackers of old, I'm totally
okay with that.
